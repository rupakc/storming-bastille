Storming Bastille is an AI research tool that maps causal chains between historical events — not just facts, but the relationships between them.

★ View on GitHub Read the Wiki →
The idea

History isn't linear. Your tools should reflect that.

You ask a question — "What led to the French Revolution?" — and within seconds Storming Bastille starts building a structured causal map: a streamed narrative backed by live web research, an interactive graph of connected events with confidence-weighted edges, a zoomable timeline, and source citations ranked by reliability. Five AI agents coordinate behind the scenes, each doing one job well. The result is a session you can explore, edit, name, and revisit — not a paragraph you copy and paste.

Features

Everything you need to understand causation

Built around a single idea: causal relationships between events matter more than chronological facts.

◈

Interactive Causal Graph

React Flow canvas with custom nodes and edges. Five relationship types — direct cause, contributing factor, enabling condition, consequence, feedback loop — each with a confidence score. Drag, edit, and export to PNG.

⟳

Real-time Streaming

Narrative text streams token by token via SSE. Graph nodes appear within 2 seconds. Causal edges arrive as a second pass. You watch the analysis build itself — never stare at a spinner.

✓

Verified Source Citations

A dedicated SourceVerifier agent cross-checks claims against scraped web sources and assigns reliability tiers: Academic, Encyclopedia, News, Web, Blog. Citations appear in a ranked panel.

◷

D3 Zoomable Timeline

Events plotted chronologically on a D3 SVG timeline. Zoomable, pannable, and linked to the graph — click a timeline event to highlight its node.

⊞

Session Memory

Name and save any analysis. Sessions persist server-side with SQLite + GCS backup. Pick up where you left off, run follow-up queries that build on prior context.

⬡

Multi-agent AI Pipeline

Five specialised agents: HistorianAgent (Claude Haiku), CausalAnalystAgent (Claude Sonnet), GraphBuilderAgent, SourceVerifierAgent, and FollowUpAgent. Each optimised for its task.

Under the hood

4-phase orchestration

Designed to minimise time-to-first-pixel. Something meaningful is on screen within 2 seconds of submitting a query.

1
Parallel · ~1–2s first token

Research & Narrative

DuckDuckGo and Bing are queried simultaneously. Results are injected as plain text into the HistorianAgent prompt — not via Claude tool-use, to avoid latency overhead. HistorianAgent (Claude Haiku 4.5) streams the narrative and embeds 8–10 structured events as JSON.

2
Immediate · ~2s

Graph Nodes

GraphBuilderAgent converts the historian's structured events into React Flow nodes and Dagre-positioned timeline data. An SSE event fires — nodes appear on screen before any causal analysis begins. No AI call needed; this is pure deterministic transformation.

3
Async · 5–15s

Causal Analysis

CausalAnalystAgent (Claude Sonnet) identifies relationships between events: type, confidence score (0.3–1.0), and explanation. A second SSE event fires with the edge set. The graph completes visually — edges appear between the existing nodes.

4
Fire-and-forget

Source Verification & Persistence

SourceVerifierAgent cross-checks claims and assigns reliability tiers. Results are saved to SQLite with a GCS backup. This runs in the background — the user has everything they need before this phase completes.

Quick start

Running locally in under 5 minutes

bash 
# Clone the repo
$ git clone https://github.com/rupakc/storming-bastille.git
$ cd storming-bastille

# Set your Anthropic API key
$ cp backend/.env.example backend/.env
$ echo "ANTHROPIC_API_KEY=sk-ant-..." >> backend/.env

# Start both backend and frontend
$ ./start.sh

# Open http://localhost:3000 — default login: admin / changeme

Prerequisites: Python 3.12+, uv, Bun. Full setup instructions in the Setup and Installation wiki page.
